**Content network architektura:** central CMS `content-network-cms-dashboard-redesign` (Next.js + Supabase) + multiple Astro content sites. Schema migrations vlastní CMS repo.

**CF Worker resource limity:**
- 1102 = "Worker exceeded resource limits" (CPU nebo memory)
- Per-request CPU max ~10ms na free plan; 50ms na paid. Lookbehind/ahead regex na velkém glossary se sčítá rychle.
- **Pravidlo**: vždy `String.includes()` prefilter před lookbehind regex. Preferuj early-exit.
- **Symptom**: HTTP 200 + 0-byte body. Zkontroluj `cf-ray` header a CF dashboard logs.

**OG image pipeline (S4.3):**
- TTF fonty z Google Fonts gstatic CDN potřebují non-browser User-Agent (jinak 404). Cache v `.og-cache/fonts/` (gitignored).
- satori nepodporuje WOFF/WOFF2 — TTF/OTF only. @fontsource ships jen WOFF/WOFF2, proto fetch z gstatic.
- @resvg/resvg-js je nativní binárka (rust+napi) — funguje na macOS/linux v dev/CI. Pro CF Worker by nešlo (Workers = isolate runtime).
- 70 images se buildí ~15s. Acceptable for prebuild step.

**HowTo schema gotchas:**
- HowTo JSON-LD je Google rich result kandidát ale typicky se UI nezobrazí pokud structured data není v souladu s viditelným obsahem stránky. Naše implementace ukazuje pronumerované kroky 1-N přesně podle frontmatter `steps[]`, takže Google by měl být happy.
- `totalTime` ISO 8601 format: "PT2H" = 2 hours, "PT30M" = 30 minutes, "PT1H30M" = 1.5h.

**Wikidata API (pro plemena + budoucí znacka enrichment):**
- `wbsearchentities` returns hits with optional `description` field (cs nebo en podle language param)
- `wbgetentities` with `props=sitelinks|descriptions&sitefilter=cswiki|enwiki` returns cs/en sitelinks pro Wikipedia URL
- Heuristic: trust top hit pokud description matchne breed/breed-related keyword via **word-boundary regex** (NE substring — "skot" by matchnul "Skotska"). Conservative = better empty than wrong.
